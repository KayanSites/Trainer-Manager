# Trainer Manager — دليل التشغيل

الملفات:
- **`app.html`** — التطبيق كله (شاشة واحدة، كل الصفحات جواها)
- **`schema.sql`** — الداتابيز
- **`README.md`** — الملف ده

## المرحلة ١ — جربي الديمو (٠ دقيقة إعداد)

افتحي `app.html` في المتصفح. خلاص.

`DEMO: true` معناها إن التطبيق شغال ببيانات وهمية في الميموري — **مفيش Supabase، مفيش حفظ**. اقفلي الصفحة وكل حاجة ترجع زي ما كانت.

جربي:
- زرار **EN / ع** فوق → اللغة
- زرار **القمر / الشمس** → لايت ودارك
- **سجّل حضور** → غيّري الجروب والتاريخ
- **إكسترا** → علّمي طالب واربطيه بغياب قديم
- **صحّح تاسكات** → افتحي (⋮) لدرجة وملاحظة
- **الإدارة** → كورس → محاضرة → تاسك → مرفقات
- **تقرير طالب** → دوسي PDF

في الديمو **يوسف** غايب في محاضرتين، وواحدة منهم اتعوّضت في حصة إكسترا — شوفي تقريره وهتلاقي الغياب لسه مسجل، وتحته "اتعوّض".

## المرحلة ٢ — Supabase

### ١. اعملي مشروع
supabase.com → New project. استنّي دقيقتين لما يخلص.

### ٢. نفّذي الـ SQL
**SQL Editor** (يسار) → **New query** → الصقي محتوى `schema.sql` كله → **Run**.

لازم تشوفي `Success`. لو ظهر خطأ، ابعتيه لي.

### ٣. اعملي حساب المدربة
**Authentication** → **Users** → **Add user** → **Create new user**

- اكتبي إيميلها وباسورد قوي
- **علّمي على `Auto Confirm User`** ← مهم، من غيرها مش هتقدر تدخل

### ٤. هاتي المفاتيح
**Project Settings** → **API**:
- `Project URL`
- `anon public` key

### ٥. عدّلي `app.html`
افتحيه بأي محرر، ودوري على بلوك `CONFIG` فوق في الـ `<script>`:

```js
const CONFIG = {
  DEMO: false,                                  // ← غيّريها لـ false
  SUPABASE_URL : 'https://xxxx.supabase.co',    // ← الـ Project URL
  SUPABASE_ANON: 'eyJhbGciOi...',               // ← الـ anon key
  ...
  LANG : 'ar',        // اللغة الافتراضية
  THEME: 'light',     // الثيم الافتراضي
};
```

احفظي، وافتحي الملف.

**هتلاقي شاشة دخول.** ادخلي بالإيميل والباسورد → التطبيق هيفتح **فاضي تمامًا** (طبيعي، لسه مفيش بيانات).

### ٦. املي البيانات
من تاب **الإدارة**، بالترتيب ده:
1. **الكورسات** → اعملي كورس
2. افتحي الكورس → ضيفي المحاضرات
3. افتحي محاضرة → ضيفي التاسكات والمرفقات
4. **الطلاب** → ضيفي الطلاب
5. **الجروبات** → اعملي جروب واربطيه بالكورس
6. افتحي الجروب → ضيفي الطلاب (كل واحد بتاريخ التحاقه)

بعد كده تقدري تبدأي تسجلي حضور.

## المرحلة ٣ — GitHub Pages

1. Repo جديد → ارفعي `app.html` (وسمّيه `index.html` لو عايزة اللينك يبقى نضيف)
2. **Settings** → **Pages** → Source: `main` / root → Save
3. بعد دقيقة يبقى شغال على `https://username.github.io/repo/`

**على الموبايل:** افتحي اللينك → Share → **Add to Home Screen**. هيبقى زي أبليكيشن.

## نقطة أمنية مهمة

الـ `anon key` **مش سر** — ده مصمم إنه يبقى في المتصفح، ومكتوب في كل مشاريع Supabase العامة. اللي بيحمي البيانات هو **RLS**، مش إخفاء المفتاح.

الـ SQL بتاعك مفعّل فيه:
- **RLS على كل الجداول** — كل صف مربوط بـ `owner_id`، ومحدش يشوف غير بتاعه
- **`security_invoker` على الـ views** — دي كانت الثغرة الحقيقية. الـ views في بوستجرس بتتخطى الـ RLS افتراضيًا، والسطور دي بتقفلها

اللي لازم **يفضل سر**: الـ `service_role` key. **متحطيهوش في `app.html` أبدًا**، ولا في أي ملف على GitHub.

## المساحة

الفري تير = **١ جيجا**. مؤشر المساحة موجود فوق في تاب الإدارة.

**القاعدة:** البوربوينت → **لينك درايف**. الصور والـ PDF الصغير → **رفع مباشر** (الصور بتتصغّر تلقائيًا لـ ١٦٠٠px وبتتحول webp، يعني صورة ٤ ميجا بتبقى ~٢٠٠ كيلو).

لو وصلتي ٧٠٪، المؤشر هيبقى أصفر. لو خلصت فعلاً، دالة `uploadFile()` معزولة في ملف واحد — تغييرها لـ Cloudflare R2 (١٠ جيجا مجانًا) بيتم من مكان واحد بس.

## قواعد النظام (اللي اتبنى عليها كل حاجة)

1. **الكورس ≠ الجروب.** الكورس محتوى. الجروب ناس ومواعيد.
2. **المحاضرة ≠ السيشن.** المحاضرة محتوى ثابت. السيشن حصة حصلت في يوم.
3. **الطالب بيتحاسب من `joined_at` بس.** طالب دخل متأخر، الحصص والتاسكات اللي قبله مش بتتحسب عليه.
4. **الإكسترا مش بتأثر على نسبة الحضور خالص.** الغياب بيفضل مسجل — عشان جايزة الالتزام آخر السنة تبقى عادلة. بس بيظهر تحته "اتعوّض".
5. **الحصة الملغية مش بتتحسب على حد.**
6. **مفيش قفل.** أي حضور أو تصحيح تقدري تعدليه في أي وقت.
7. **الحذف soft.** الطالب بيختفي من الشاشات بس تاريخه محفوظ.
